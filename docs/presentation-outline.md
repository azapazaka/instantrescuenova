## Слайд 2. Проблема

**Три конкретные точки отказа, а не «улучшить здравоохранение»:**

1. **Ухудшение замечают поздно.** Мерцательная аритмия часто бессимптомна.
   Человек узнаёт о ней после инсульта. Между приёмами пульс никто не смотрит.
2. **Анализы непонятны.** PDF с цифрами → человек ждёт врача, чтобы понять.
3. **Падение = потерянные часы.** Падение может быть следствием инсульта.

**Контекст:** ССЗ — 32% всех смертей в мире, 19,8 млн человек в 2022 (ВОЗ).
Большинство преждевременных смертей предотвратимы ранним выявлением.

> Говорить голосом человека: «Наша бабушка живёт одна. Мы узнаём о проблеме
> последними.»

## Слайд 3. Пользователь

**Кто:** пожилой человек 65+, живёт один или частично самостоятельно.
**Второй пользователь:** близкий, который получает уведомления.

**Сценарий за 20 секунд:**
открыл → увидел пульс → система объяснила, что изменилось → при падении близкий
получил Telegram с временем и пульсом.

**Почему UI такой:** текст от 18px, кнопки от 48px, статус никогда не передаётся
только цветом (иконка + слово), потому что пользователь — пожилой человек с
телефоном на солнце.

## Слайд 4. Решение (демо)

Показывать вживую, не скриншотами:

1. Живой пульс на главной.
2. Переключить сценарий → **«Мерцательная аритмия»**.
3. График на глазах становится рваным, оценка → «Отклонение».
4. Панель «Что увидела модель»: pNN50 — вклад 79%, «выше вашей нормы на 63%».
5. План дня → рекомендации со **ссылками на ВОЗ/AHA**.
6. «Проверить падение» → сообщение приходит в Telegram на телефон.

## Слайд 5. Данные

| Источник | Что берём |
|---|---|
| MIT-BIH Arrhythmia DB (PhysioNet, ODC-BY) | RR-интервалы + разметка ритма, 48 пациентов |
| ВОЗ / AHA / ESC | 16 выдержек в RAG-корпус, вручную из первоисточников |
| Симулятор | живой поток для демо (помечен в UI и в БД) |

**Ключевое решение:** берём **не сырой сигнал ЭКГ, а RR-интервалы** — то, что
реально выдаёт бытовой пульсометр. Модель на форме волны 360 Гц на браслет бы не
перенеслась.

**1639 окон, 48 пациентов, 16,7% аномальных.**

## Слайд 6. AI/ML-подход

**Три механизма, потому что задачи разные:**

| Что | Чем | Почему не иначе |
|---|---|---|
| Нерегулярность ритма | IsolationForest + GradientBoosting | Правила так не умеют |
| Частота пульса (тахи/бради) | Клинические правила | В MIT-BIH 6 окон тахикардии из 1639 — модель этому не научится |
| Совет и объяснение | Groq LLM + RAG | Нужен язык и ссылки на источники |

**Метрики на 15 пациентах, которых модель не видела:**

```
IsolationForest    ROC-AUC 0.881
GradientBoosting   ROC-AUC 0.901
abnormal:  precision 0.668   recall 0.888   f1 0.763
accuracy   0.825
```

**Два инженерных решения, которые стоит проговорить:**

- **Разделение по пациентам, а не случайное.** Случайное дало бы «почерк» сердца
  одного человека и в train, и в test — метрики были бы враньём.
- **Порог 0.05, а не 0.5.** Это скрининг: пропустить аритмию хуже, чем лишний раз
  отправить к врачу. Порог по максимуму F2. При 0.5 recall был бы **0.503** —
  половина аномалий пропущена. Мы сознательно платим точностью за полноту.

## Слайд 7. Explainability

«Anomaly score 0.71» бесполезен для пожилого человека. Что показываем:

- **Какие данные повлияли** — вклад каждого признака в %.
- **Насколько** — «pNN50 выше вашей нормы на 63% (75 против 12)».
- **С чем сравнили** — «норма здорового взрослого» / «ваша личная норма».
- **Где ошибаемся** — «модель видит только пульс, не знает о лекарствах».
- **Кто решает** — «итоговое решение за вами и вашим врачом».

**Против выдуманных ссылок:** модель получает выдержки с `id` и возвращает
`source_ids`. Всё, чего не было в корпусе, отбрасывается. Сослаться на
несуществующий документ она физически не может.

## Слайд 8. Ограничения (говорить честно — это плюс)

- MIT-BIH — **не пожилые люди**, и база специально насыщена аритмиями.
- RR с экспертной разметки чище, чем с оптического датчика на запястье.
- **Найдено при тестировании:** популяционная база MIT-BIH не годится для
  объяснения здоровому пользователю — её «норма» взята у аритмических пациентов
  (pNN50 ~26% против ~12%). Поэтому объяснения считаются от отдельной здоровой
  базовой линии.
- Пульс в демо — симуляция, помечена в интерфейсе и в БД.
- Precision 0.668 → ложные срабатывания есть. Это осознанная цена за recall.
- Система не ставит диагноз и не заменяет врача.

## Слайд 10. Стек и запуск

**Инфраструктура:** Supabase (Auth ES256/JWKS + Postgres + RLS)

**59 тестов, RLS на всех таблицах, PDF не сохраняется, токен устройства — только хеш.**

```bash
cd backend && pip install -r requirements.txt && uvicorn app.main:app --reload
cd frontend && npm install && npm run dev
```
